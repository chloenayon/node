• NODE



Node was originally written in 2009 by Ryan Dahl. It's written in C and C++, implements V8 (a Javascript execution engine created for Google Chrome), and uses Javascript syntax.


It is an open source, event-driven, asynchronous, non-blocking environment best used for developing dynamic, real-time server-side application.


Asynchronous and non-blocking mean the same thing. A synchronous operation blocks a process until the entire operation is complete. But an asynchronous operation merely initiates the process. Node.js implements a callback method in which a single thread is used as various events trigger requests. Instead of waiting for the single operation to complete, which could drain time and efficiency, Node can process a high number of requests without waiting for a function to return a result.


PROS
  • dynamic and efficient
  • highly scalable
  • very fast
  • same code shared with both server and client side
CONS
  • unsuited for CPU-intensive tasks
  • relatively new, still in development
  • incompatible with relational databases

EXAMPLE: SIMPLE SERVER

var http = require('http');
http.createServer(function (req, res) {
res.writeHead(200, {'Content-Type': 'text/plain'});
res.end('Hello World\n');
}).listen(8000);

Step Explanation


<- creates server and request handler
<- sets status code and writes headers
<- writes actual content
<- runs on port 8000
